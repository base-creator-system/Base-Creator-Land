# Contributing

## Purpose

This document defines the contribution standards for the BASE Tier 2 Creator Template v1.

Its purpose is to preserve the integrity, consistency, and long-term evolution of the BASE Creator System.

---

# Contribution Philosophy

Every contribution should strengthen the BASE Tier 2 Creator Template v1.

Contributions should improve the system without introducing unnecessary complexity, regression, or architectural drift.

---

# Development Standards

Contributors should:

- Preserve system integrity.
- Preserve deployment portability.
- Preserve creator independence.
- Preserve documentation accuracy.
- Maintain compatibility whenever practical.
- Keep implementations consistent throughout the system.

---

# Documentation

Documentation is considered part of the product.

When functionality changes:

- Documentation should be updated.
- Version information should be updated.
- Changelog entries should be updated when appropriate.

---

# Architecture

Contributions should respect the established architecture of the BASE Creator System.

Changes should not:

- Introduce unnecessary architectural drift.
- Rename established terminology without justification.
- Remove existing functionality without documented reason.
- Introduce unnecessary dependencies.

---

# Testing

Before contributing changes, verify:

- Creator Dashboard functionality.
- Fan App functionality.
- Desktop operation.
- Mobile operation.
- Progressive Web App (PWA) behavior.
- Publishing workflow.
- Deployment workflow.

---

# Releases

Production releases should not include unverified functionality.

Every official release should represent a stable production milestone.

---

# Contribution Goal

Every contribution should improve the BASE Tier 2 Creator Template v1 while preserving the principles, stability, and long-term vision of the BASE Creator System.
