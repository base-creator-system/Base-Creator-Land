# BASE Builder Guide

## Purpose

This guide defines the standard workflow for building Creator Deployments using the BASE Tier 2 Creator Template v1.

Its purpose is to provide a consistent, repeatable process while preserving the integrity of the BASE Creator System.

---

# Builder Philosophy

The BASE Creator System provides the software.

The Builder creates the Creator Deployment.

Every Creator Deployment should preserve the creator's identity while maintaining the integrity of the BASE Creator System.

System improvements belong in future releases of the BASE Tier 2 Creator Template.

Creator customizations belong within the Creator Deployment.

---

# Standard Builder Workflow

1. Start with the official BASE Tier 2 Creator Template v1.
2. Configure the Creator Dashboard.
3. Customize the Creator Deployment.
4. Publish the deployment.
5. Verify desktop operation.
6. Verify mobile operation.
7. Verify Progressive Web App (PWA) installation.
8. Validate deployment behavior.
9. Freeze the deployment.
10. Release the deployment.

---

# Creator Customization

Each Creator Deployment may customize:

- Creator Identity
- Creator Branding
- Creator Logo
- Creator Hero
- Creator Background
- Creator Theme
- Creator Description
- Creator Media
- Creator Links
- Creator Community
- Creator Live Experience
- Creator Supporters
- Creator Contact Information
- Creator External Media

---

# System Preservation

The BASE Creator System should remain consistent across Creator Deployments.

System enhancements should be introduced through future releases of the BASE Tier 2 Creator Template rather than through inconsistent deployment modifications.

---

# Hosting Independence

The BASE Creator System is independent of any hosting provider.

Builders are free to deploy Creator Deployments using any compatible hosting environment.

Hosting providers are infrastructure.

They do not define the BASE Creator System.

Moving a deployment between hosting providers does not change the identity of the BASE Creator System.

---

# Privacy by Design

The BASE Creator System is designed to operate without requiring user accounts, audience tracking, or system-level data collection.

Builders may integrate third-party services when appropriate.

Those services operate independently of the BASE Creator System and are governed by their own functionality and policies.

---

# Version Management

Each Creator Deployment should record the version of the BASE Tier 2 Creator Template v1 used during its creation.

Future releases of the BASE Tier 2 Creator Template may be adopted at the creator's discretion.

---

# Builder Goal

Every Creator Deployment should begin from the official BASE Tier 2 Creator Template v1 while allowing each creator to build an independent experience without compromising the integrity, portability, or long-term evolution of the BASE Creator System.
