# BASE Builder Guide

## Purpose

This guide defines the standard workflow for building Creator Deployments using the BASE Creator System.

Its purpose is to provide a consistent, repeatable process while preserving the integrity of the BASE Framework.

---

# Builder Philosophy

BASE provides the framework.

The builder creates the Creator Deployment.

Every Creator Deployment should preserve the creator's identity while maintaining the integrity of the BASE Framework.

Framework improvements belong in future BASE Framework releases.

Creator customizations belong within the Creator Deployment.

---

# Standard Builder Workflow

1. Start with the official BASE Creator Template.
2. Configure the Creator Dashboard.
3. Customize the Creator Deployment.
4. Publish the deployment.
5. Verify desktop operation.
6. Verify mobile operation.
7. Verify Progressive Web App (PWA) installation.
8. Validate deployment behavior.
9. Freeze the deployment.
10. Release the deployment.

---

# Creator Customization

Each Creator Deployment may customize:

- Creator Identity
- Creator Branding
- Creator Logo
- Creator Hero
- Creator Background
- Creator Theme
- Creator Description
- Creator Media
- Creator Links
- Creator Community
- Creator Live Experience
- Creator Supporters
- Creator Contact Information
- Creator External Media

---

# Framework Preservation

The BASE Framework should remain consistent across Creator Deployments.

Framework enhancements should be developed as future BASE Framework releases rather than through inconsistent deployment modifications.

---

# Hosting Independence

The BASE Framework is independent of any hosting provider.

Builders are free to deploy Creator Deployments using any compatible hosting environment.

Hosting providers are infrastructure.

They do not define the BASE Framework.

Moving a deployment between hosting providers does not change the identity of the BASE Framework.

---

# Privacy by Design

The BASE Framework is designed to operate without requiring user accounts, audience tracking, or framework-level data collection.

Builders may integrate third-party services when appropriate.

Those services operate independently of the BASE Framework and are governed by their own functionality and policies.

---

# Version Management

Each Creator Deployment should record the BASE Framework version used during its creation.

Future BASE Framework releases may be adopted at the creator's discretion.

---

# Builder Goal

Every Creator Deployment should begin from the official BASE Creator Template while allowing each creator to build an independent experience without compromising the integrity, portability, or long-term evolution of the BASE Framework.
