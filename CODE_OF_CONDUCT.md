# Code of Conduct

## Purpose

This Code of Conduct establishes the expected standards of professionalism, collaboration, and respect for anyone contributing to the BASE Tier 2 Creator Template v1.

The objective is to encourage productive collaboration while protecting the integrity of the BASE Creator System.

---

# Professional Conduct

Contributors are expected to:

- Act professionally.
- Communicate respectfully.
- Remain constructive during technical discussions.
- Focus on improving the system.

Technical disagreements should be resolved through discussion, testing, documentation, and objective evaluation.

---

# Respect for the Project

Contributors should respect:

- The established architecture.
- The documented terminology.
- The documented release process.
- The documented contribution process.
- Existing functionality unless changes are justified and documented.

---

# Collaboration

Contributors are encouraged to:

- Share ideas openly.
- Document proposed improvements.
- Identify issues responsibly.
- Test changes before submission.
- Update documentation when functionality changes.

---

# Project Integrity

Contributors should avoid:

- Intentional regression.
- Unnecessary architectural drift.
- Undocumented breaking changes.
- Misrepresentation of project ownership.
- Removal of copyright or licensing notices.

---

# Inclusion

Constructive contributions are welcome regardless of background, experience level, or geographic location.

Professional collaboration strengthens the long-term evolution of the BASE Creator System.

---

# Enforcement

Project maintainers reserve the right to reject, request revision of, or decline contributions that conflict with this Code of Conduct or compromise the integrity of the BASE Creator System.

---

# Goal

The goal of this Code of Conduct is to support a professional, respectful, and sustainable development environment that preserves the quality and long-term vision of the BASE Creator System.
