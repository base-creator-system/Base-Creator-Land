# Security Policy

## Purpose

This document defines the security reporting process for the BASE Tier 2 Creator Template v1.

The objective is to encourage responsible reporting while protecting users, Creator Deployments, and the integrity of the BASE Creator System.

---

# Responsible Disclosure

If you discover a security issue affecting the BASE Tier 2 Creator Template v1, please report it responsibly.

Public disclosure should be avoided until the issue has been reviewed and, when appropriate, resolved.

---

# Reportable Issues

Examples include:

- Security vulnerabilities
- Unauthorized access risks
- Deployment vulnerabilities
- Publishing workflow vulnerabilities
- Authentication or authorization issues
- Data exposure risks
- Service worker vulnerabilities
- Progressive Web App (PWA) security concerns
- Third-party integration security concerns

---

# Reporting Guidelines

Security reports should include, when possible:

- Description of the issue
- Steps to reproduce
- Affected version
- Expected behavior
- Actual behavior
- Potential impact
- Supporting screenshots, logs, or proof of concept when appropriate

---

# Investigation

Each reported issue should be reviewed to determine:

- Scope
- Severity
- Reproducibility
- Potential impact
- Appropriate resolution

Security updates should be documented when applicable.

---

# Coordinated Resolution

When practical, verified security issues should be resolved before public disclosure.

Resolved issues may be documented in future release notes or changelog entries when appropriate.

---

# Third-Party Services

Security issues affecting third-party services remain the responsibility of their respective providers.

Only issues directly affecting the BASE Tier 2 Creator Template v1 should be reported under this policy.

---

# Security Goal

The goal of this Security Policy is to encourage responsible reporting, timely resolution of verified security issues, and the continued protection of the BASE Creator System and Creator Deployments.
